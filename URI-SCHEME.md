# Tamizdat URI Scheme

Tamizdat clients can be configured with a URI generated by the admin panel. Public documentation should use placeholders only.

## Shape

```text
tamizdat://<short-id>@server.example.com:443/?sni=cover.example.com&pubkey=<server-public-key-hex>&fp=mix
```

Fields:

- `server.example.com:443` — server address and port.
- `<short-id>` — per-user short identifier generated by the panel.
- `sni` — TLS Server Name used for the cover connection.
- `pubkey` — server X25519 public key in hexadecimal form.
- `fp` — client TLS fingerprint profile.
- `transport` — optional transport selector; omit it for default H2 mode. Advanced deployments may use values such as `vkturn` when a compatible TURN relay profile is available.

## Safety

Treat complete URIs as credentials. Do not publish real URIs in issues, documentation, screenshots, logs, or examples.

## Transport notes

Default profile URIs should use H2 implicitly or with `transport=h2`. TURN-style profiles are deployment-specific and should only be generated when the server/client pair and relay credentials are known to work.
