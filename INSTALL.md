# Installation

This guide installs Tamizdat server, admin panel, Linux client binary, and the `tamizdat` management command on a Linux host with systemd.

## Requirements

Runtime installation:

- Linux with systemd
- Root access
- Python 3.9 or newer
- OpenSSL, CA certificates, and `tar`
- A public DNS name pointing to the server if clients connect by hostname

Building release binaries from source additionally requires Go 1.25 or newer.

Examples below use placeholders:

- `server.example.com` — your server hostname
- `cover.example.com` — a neutral TLS cover hostname
- `443` — default Tamizdat server port
- `<random>` — panel port/base path generated by default

Replace them with values for your deployment.

## Easy install from GitHub Releases

Tagged release / prerelease install:

```bash
curl -fsSL https://github.com/funnybones69/tamizdat/releases/download/<tag>/install.sh -o install.sh
sudo bash install.sh
```

Use the exact tag from the GitHub Releases page, for example `v0.1.0-preview.5`.

After a stable non-prerelease is published, `/releases/latest/download/install.sh` can be used. Tagged `install.sh` assets can be baked with an exact release tag. If you build your own prerelease, set `TAMIZDAT_BAKE_RELEASE_BASE` while packaging or run with `TAMIZDAT_RELEASE_BASE=https://github.com/<owner>/<repo>/releases/download/<tag>`.

The standalone installer detects CPU architecture and downloads:

```text
tamizdat-linux-<arch>.tar.gz
```

from the latest GitHub Release. The release tarball contains:

```text
tamizdat/
  tamizdat-server-app
  tamizdat-client
  tamizdat-panel.py
  tamizdat
  install.sh
  uninstall.sh
  README.md
  INSTALL.md
  OPENWRT.md
  LICENSE
```

The OpenWrt/LuCI router client is published separately as:

```text
tamizdat-openwrt-luci-aarch64-20260622.tar
```

See [OPENWRT.md](OPENWRT.md) for router installation, LuCI setup, and policy-routing notes.

## Install from a downloaded release bundle

```bash
tar xf tamizdat-linux-amd64.tar.gz
cd tamizdat
sudo ./install.sh
```

The installer will:

1. install runtime dependencies;
2. locate bundled files or download the release bundle;
3. install files into `/usr/local/tamizdat/`;
4. create compatibility symlinks in `/usr/local/bin/`;
5. install the management command as `/usr/bin/tamizdat`;
6. create `/etc/tamizdat/`;
7. generate server key material if needed;
8. ask for the public hostname/IP used in generated client URIs;
9. create or update two systemd units;
10. bind the admin panel to `127.0.0.1` by default;
11. start the services;
12. print the panel URL and new admin credentials when generated/reset.

Save newly printed admin credentials immediately. Existing installs keep the current admin password unless you explicitly reset it.

## Installed layout

```text
/usr/local/tamizdat/
  bin/
    tamizdat-server-app
    tamizdat-client
  panel/
    tamizdat-panel.py
  scripts/
    tamizdat
    uninstall.sh

/usr/bin/tamizdat                         # management command
/usr/local/bin/tamizdat-server-app         # compatibility symlink
/usr/local/bin/tamizdat-client             # compatibility symlink
/usr/local/bin/tamizdat-panel.py           # compatibility symlink

/etc/tamizdat/
  data.db
  inbound_priv_key.hex
  shortid.hex
  cert.pem
  key.pem
  install-info.txt
```

Systemd units remain separate by design:

```text
tamizdat-server-app.service
tamizdat-panel.service
```

## Management command

```bash
tamizdat status
tamizdat panel-url
tamizdat logs
tamizdat logs server
tamizdat logs panel
tamizdat restart
tamizdat version
```

The command avoids printing secrets. Client profile URIs and short IDs should still be handled through the panel and kept private.

## Build a release bundle from source

From a source checkout:

```bash
go test ./...
python3 -m unittest panel/test_panel_shortid.py
python3 -m unittest scripts/test_install.py

GOARCH=amd64 ./scripts/package-linux.sh
GOARCH=arm64 ./scripts/package-linux.sh
./scripts/package-windows.sh
```

This writes:

```text

dist/tamizdat-linux-amd64.tar.gz
dist/tamizdat-linux-arm64.tar.gz
dist/tamizdat-windows-amd64.zip
dist/SHA256SUMS
```

Upload `install.sh`, `tamizdat-linux-<arch>.tar.gz`, `tamizdat-windows-amd64.zip`, and `SHA256SUMS` to the same GitHub Release. The standalone `install.sh` downloads `tamizdat-linux-<arch>.tar.gz` from that release.

Verify downloaded assets:

```bash
sha256sum -c SHA256SUMS
```

```bash
tar xf tamizdat-linux-amd64.tar.gz
cd tamizdat
sudo ./install.sh
```

## Non-interactive install

For automation, set the variables used by `install.sh`:

```bash
sudo TAMIZDAT_INSTALL_NONINTERACTIVE=1 \
  TAMIZDAT_INSTALL_PORT=8888 \
  TAMIZDAT_INSTALL_USERNAME=admin \
  TAMIZDAT_INSTALL_PASSWORD='<strong-panel-password>' \
  TAMIZDAT_INSTALL_BIND_ADDR=127.0.0.1 \
  TAMIZDAT_PANEL_SERVER_HOST=server.example.com \
  TAMIZDAT_VPN_PORT=8443 \
  TAMIZDAT_LISTEN_ADDR=0.0.0.0 \
  ./install.sh
```

Useful installer variables:

- `TAMIZDAT_APP_DIR` — app directory; defaults to `/usr/local/tamizdat`.
- `TAMIZDAT_INSTALL_PORT` — admin panel port.
- `TAMIZDAT_INSTALL_BASE_PATH` — admin panel URL path; random when unset.
- `TAMIZDAT_INSTALL_USERNAME` — admin username; defaults to `admin`.
- `TAMIZDAT_INSTALL_PASSWORD` — admin password; random on fresh install when unset. Existing installs keep the current admin password unless this is set or `TAMIZDAT_INSTALL_RESET_ADMIN=1` is used.
- `TAMIZDAT_INSTALL_BIND_ADDR` / `TAMIZDAT_PANEL_BIND_ADDR` — panel bind address; defaults to `127.0.0.1`.
- `TAMIZDAT_PANEL_SERVER_HOST` / `TAMIZDAT_INSTALL_HOSTNAME` — hostname or IP embedded into generated client URIs; defaults to `hostname -f` then the primary local IP.
- `TAMIZDAT_VPN_PORT` — server inbound port; defaults to `443`.
- `TAMIZDAT_LISTEN_ADDR` — server listen address; defaults to `0.0.0.0`.
- `TAMIZDAT_INSTALL_CLIENT=0` — skip installing the Linux client binary.
- `TAMIZDAT_SERVER_BIN_SRC` — local path to a server binary to install.
- `TAMIZDAT_CLIENT_BIN_SRC` — local path to a client binary to install.
- `TAMIZDAT_PANEL_SRC` — local path to `tamizdat-panel.py`.
- `TAMIZDAT_MANAGER_SRC` — local path to the `tamizdat` management command.
- `TAMIZDAT_RELEASE_BASE` — alternate release download base URL.
- `TAMIZDAT_BAKE_RELEASE_BASE` — packaging-time value baked into `dist/install.sh` and Linux tarball installers for prerelease/tag-specific assets.

## Firewall

Open the Tamizdat server port for clients. Default install uses `443/tcp`:

```bash
sudo ufw allow 443/tcp
```

If you set `TAMIZDAT_VPN_PORT=8443`, open that port instead:

```bash
sudo ufw allow 8443/tcp
```

Keep the panel private when possible. The installer defaults to `127.0.0.1`; this is intentional. The raw panel speaks HTTP on localhost and should not be exposed directly to the public Internet. Use one of these access patterns:

- SSH forwarding for private/admin-only access;
- a private management network;
- an HTTPS reverse proxy with a real certificate and optional extra authentication.

Get the current panel URL:

```bash
tamizdat panel-url
```

Example SSH tunnel when the panel listens on `127.0.0.1:8888`:

```bash
ssh -L 127.0.0.1:8888:127.0.0.1:8888 root@server.example.com
```

Bind the local side explicitly to `127.0.0.1`. On hosts without a working IPv6 loopback, the shorter `ssh -L 8888:127.0.0.1:8888 ...` form can fail while trying to bind `::1`.

Then open the URL printed by `tamizdat panel-url` locally, replacing the host with `127.0.0.1` if needed.

## Panel HTTPS certificate / reverse proxy

The recommended public setup is TLS termination in nginx/Caddy/another reverse proxy. Do not put TLS certificates into the panel process itself unless you are intentionally developing a different deployment mode.

Important port choice:

- By default the Tamizdat protocol server uses public TCP `443`.
- A public HTTPS panel on the same host also wants `443`.
- One process cannot own the same public IP/port twice.

Choose one:

1. **Private panel, protocol on 443** — recommended. Keep the panel on `127.0.0.1` and use SSH tunnel.
2. **Public panel on 443** — install Tamizdat protocol on another port, for example `TAMIZDAT_VPN_PORT=8443`, then let nginx own `443` for the panel.
3. **Protocol on 443 and panel HTTPS on another port** — keep Tamizdat on `443`, run nginx for the panel on a separate TLS port such as `8444`.

Example: panel HTTPS on standard `443`, Tamizdat protocol moved to `8443`:

```bash
sudo TAMIZDAT_INSTALL_NONINTERACTIVE=1 \
  TAMIZDAT_PANEL_SERVER_HOST=server.example.com \
  TAMIZDAT_VPN_PORT=8443 \
  ./install.sh

sudo apt-get update
sudo apt-get install -y nginx certbot python3-certbot-nginx apache2-utils
```

Create optional extra HTTP Basic Auth for the panel:

```bash
sudo htpasswd -c /etc/nginx/.htpasswd-tamizdat admin
```

Get panel backend values:

```bash
tamizdat panel-url
# Example output: http://127.0.0.1:8888/abyss-random/
```

First create a temporary HTTP-only nginx vhost so certbot can validate the domain. `/etc/nginx/sites-available/tamizdat-panel.conf`:

```nginx
server {
    listen 80;
    server_name panel.example.com;

    location / {
        return 200 "tamizdat panel certificate bootstrap\n";
    }
}
```

Enable it and issue the certificate. Make sure DNS for `panel.example.com` points to this server and TCP `80` is reachable:

```bash
sudo ln -sf /etc/nginx/sites-available/tamizdat-panel.conf /etc/nginx/sites-enabled/tamizdat-panel.conf
sudo nginx -t
sudo systemctl reload nginx
sudo certbot certonly --nginx -d panel.example.com
```

After certbot succeeds, replace `/etc/nginx/sites-available/tamizdat-panel.conf` with the real HTTPS reverse proxy. Replace the port/base path with your `tamizdat panel-url` output:

```nginx
server {
    listen 80;
    server_name panel.example.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name panel.example.com;

    ssl_certificate     /etc/letsencrypt/live/panel.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/panel.example.com/privkey.pem;

    # Optional extra gate in front of the panel login.
    auth_basic "Tamizdat panel";
    auth_basic_user_file /etc/nginx/.htpasswd-tamizdat;

    location = / {
        return 404;
    }

    location ^~ /abyss-random/ {
        proxy_pass http://127.0.0.1:8888;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto https;
    }
}
```

Validate and reload nginx:

```bash
sudo nginx -t
sudo systemctl reload nginx
```

Open:

```text
https://panel.example.com/abyss-random/
```

If Tamizdat protocol remains on `443`, use the same nginx block on another TLS port instead, for example `listen 8444 ssl http2;`, and open `https://panel.example.com:8444/abyss-random/`.


## Create the first user

1. Open the panel through the local bind address, SSH tunnel, or HTTPS reverse proxy.
2. Log in with the generated/admin credentials.
3. Create a user.
4. Copy the generated profile URI or the individual profile fields.

Never publish real generated URIs, short IDs, pubkeys, panel passwords, databases, or logs.

## Transport modes: H2 and TURN

H2 is the normal/default transport mode. It uses TLS 1.3 plus HTTP/2 CONNECT after Tamizdat profile authentication. Start with H2 unless you are intentionally testing a restricted-network fallback.

TURN relay mode is optional and deployment-specific. It is meant for networks where the direct H2 path is unreliable, blocked, or where an operator wants a relay-style fallback. TURN mode needs compatible server/client support and TURN credentials/profile data; do not publish those values.

Linux CLI H2 example:

```bash
tamizdat-client \
  -transport h2 \
  -server server.example.com:443 \
  -servername cover.example.com \
  -pubkey <server-public-key-hex> \
  -shortid <user-short-id-hex> \
  -listen 127.0.0.1:1080
```

Linux CLI TURN-style relay example with operator-supplied credentials:

```bash
tamizdat-client \
  -transport vkturn \
  -server server.example.com:443 \
  -shortid <user-short-id-hex> \
  -vk-turn-user <turn-username> \
  -vk-turn-pass <turn-password> \
  -vk-turn-urls '<turn-server-urls>' \
  -listen 127.0.0.1:1080
```

For Windows users, the panel-generated `tamizdat://` profile controls the transport used by the tray client. Keep the normal H2 profile as the first/default profile unless you have verified TURN on that network.

## OpenWrt / LuCI router client

For OpenWrt aarch64 routers, use the separate release asset:

```text
tamizdat-openwrt-luci-aarch64-20260622.tar
```

Quick install from a PC:

```bash
curl -L -o tamizdat-openwrt-luci-aarch64-20260622.tar \
  https://github.com/funnybones69/tamizdat/releases/download/<tag>/tamizdat-openwrt-luci-aarch64-20260622.tar
tar xf tamizdat-openwrt-luci-aarch64-20260622.tar
scp -O -r openwrt-tamizdat root@192.168.10.1:/tmp/
ssh root@192.168.10.1 'cd /tmp/openwrt-tamizdat && sh install.sh'
```

If the router reports missing dependencies, install them on OpenWrt and rerun the installer:

```sh
opkg update
opkg install kmod-tun nftables ip-full jsonfilter resolveip
cd /tmp/openwrt-tamizdat
sh install.sh
```

After installation open LuCI:

```text
Services → Tamizdat VPN
```

Basic flow:

1. **Servers (URIs)** — paste one or more panel-generated `tamizdat://` profiles.
2. **Routing & Settings** — select interfaces to tunnel, for example `lan`.
3. **Routing & Settings** — optionally refresh geodata and add proxy/direct rules.
4. **Status** — **Test all**, **Make active**, then **Enable & start**.

Full router guide: [OPENWRT.md](OPENWRT.md).

## Run clients

### Linux CLI client

The Linux client is a local SOCKS5 proxy. If the client machine is only a client, install the client binary from the release tarball:

```bash
tar xf tamizdat-linux-amd64.tar.gz
sudo install -m 0755 tamizdat/tamizdat-client /usr/local/bin/tamizdat-client
```

If the machine already runs a Tamizdat server/panel, do **not** blindly run the server installer again just to get a client. Extract the release tarball into a separate directory and use only `tamizdat-client`, for example:

```bash
mkdir -p /opt/tamizdat-client
sudo tar -xzf tamizdat-linux-amd64.tar.gz -C /opt/tamizdat-client --strip-components=1 tamizdat/tamizdat-client
sudo chmod 0755 /opt/tamizdat-client/tamizdat-client
```

Prefer a panel-generated profile URI file so the short ID and public key are not exposed in process arguments:

```bash
install -m 0600 panel-profile.uri ./profile.uri
tamizdat-client -config-file ./profile.uri -listen 127.0.0.1:1080
```

Equivalent explicit-field H2 form:

```bash
tamizdat-client \
  -transport h2 \
  -server server.example.com:443 \
  -servername cover.example.com \
  -pubkey <server-public-key-hex> \
  -shortid <user-short-id-hex> \
  -listen 127.0.0.1:1080
```

Then configure TCP applications to use remote-DNS SOCKS:

```text
socks5h://127.0.0.1:1080
```

The same SOCKS listener supports SOCKS5 UDP ASSOCIATE for UDP-capable clients. For example, DNS tools or applications that implement SOCKS5 UDP can send UDP through the relay after opening a UDP ASSOCIATE control connection to `127.0.0.1:1080`.

Optional local SOCKS authentication:

```bash
tamizdat-client \
  -config-file ./profile.uri \
  -listen 127.0.0.1:1080 \
  -auth-user localuser \
  -auth-pass '<local-socks-password>'
```

For a long-running Linux desktop/session, wrap that command in your user service manager, supervisor, or shell autostart. Keep profile URI files private (`0600`) and prefer loopback-only listeners unless you add local SOCKS authentication.

### Windows tray client

Download and extract:

```text
tamizdat-windows-amd64.zip
```

The zip contains one public Windows client executable:

```text
tamizdat-tray.exe
```

The TUN engine and Wintun DLL are embedded inside that executable. To run it:

1. Put `tamizdat-tray.exe` in a writable folder, for example `C:\Users\<you>\Desktop\tamizdat-tray\`.
2. Copy `config.example.uri` to `config.uri` beside the exe.
3. Replace the example line with one generated `tamizdat://` profile URI from the panel. Use one URI per non-empty line.
4. Run `tamizdat-tray.exe` as Administrator.
5. Use the tray menu to connect, disconnect, switch servers, show logs, or exit.

Tray files beside the exe:

```text
config.uri             # your private profile URI list; do not publish
tamizdat-tray.state    # last selected profile hash, not the URI
tamizdat-tray.log      # filtered/sanitized log; rotates at 5 MiB, no .1/.2 backups
```

The tray log suppresses routine TUN chatter and redacts full `tamizdat://` URIs, pubkeys, and short IDs before writing.

## Upgrade

From a new release bundle:

```bash
tar xf tamizdat-linux-amd64.tar.gz
cd tamizdat
sudo ./install.sh
tamizdat status
```

Back up `/etc/tamizdat/` first when upgrading important deployments. Re-running `install.sh` keeps the current panel admin password by default and reuses values from `/etc/tamizdat/install-info.txt` where possible. To intentionally reset the admin password during upgrade, set `TAMIZDAT_INSTALL_RESET_ADMIN=1` or provide `TAMIZDAT_INSTALL_PASSWORD`.

## Uninstall

```bash
sudo tamizdat uninstall
```

Equivalent stable path after one-line install:

```bash
sudo /usr/local/tamizdat/scripts/uninstall.sh
```

To remove config/keys/DB as well:

```bash
sudo tamizdat uninstall --purge
```

## Troubleshooting

- Client cannot connect: verify DNS, firewall, server port, and service status.
- Panel cannot log in: verify panel service logs and credentials.
- Panel is unreachable remotely: by default it binds to `127.0.0.1`; use SSH forwarding or set an explicit private bind address.
- Profile works on one network but not another: verify SNI/cover hostname, routing rules, and whether the destination network blocks the selected port.
- Do not paste full profile URIs into public issues; redact short IDs and keys.
