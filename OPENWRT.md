# OpenWrt / LuCI client

Tamizdat also ships an OpenWrt/LuCI client archive for aarch64 routers. It turns an OpenWrt router into a transparent Tamizdat client: LAN devices can use the router normally, while selected interfaces and destinations are policy-routed through Tamizdat.

The OpenWrt package is distributed as a release asset:

```text
tamizdat-openwrt-luci-aarch64-20260622.tar
```

It contains:

```text
openwrt-tamizdat/
  install.sh                         on-router installer
  README.md                          OpenWrt-specific detailed guide
  tamizdat/                          base package: binaries, procd service, UCI config, routing scripts
  luci-app-tamizdat/                 LuCI UI package
  tamizdat/prebuilt/aarch64/         static aarch64 binaries
```

The prebuilt binaries target **OpenWrt aarch64 / ARM64** routers, especially mediatek/filogic-class devices.

## What the LuCI app provides

- Store multiple `tamizdat://` profile URIs.
- Select one active profile.
- Test RTT per profile through a real Tamizdat tunnel, not just ping.
- Choose which router interfaces should be tunneled, for example `lan`.
- Split routing by destination: proxy or direct by geosite, geoip, domain/keyword, or IP/CIDR.
- Manage the service from LuCI: status, test, enable/start, stop/restart.

## Install on a running OpenWrt router

Download the archive on your PC from the release page:

```bash
curl -L -o tamizdat-openwrt-luci-aarch64-20260622.tar \
  https://github.com/funnybones69/tamizdat/releases/download/<tag>/tamizdat-openwrt-luci-aarch64-20260622.tar
```

Extract it locally:

```bash
tar xf tamizdat-openwrt-luci-aarch64-20260622.tar
```

Copy the extracted tree to the router:

```bash
scp -O -r openwrt-tamizdat root@192.168.10.1:/tmp/
```

Run the installer on the router:

```bash
ssh root@192.168.10.1 'cd /tmp/openwrt-tamizdat && sh install.sh'
```

The installer copies binaries and helper scripts, installs the procd service, installs the LuCI views/menu/ACL files, wires the TUN device into the firewall, reloads `rpcd`, and enables the Tamizdat service.

If dependencies are missing, install them first on the router:

```sh
opkg update
opkg install kmod-tun nftables ip-full jsonfilter resolveip
```

Then rerun:

```sh
cd /tmp/openwrt-tamizdat
sh install.sh
```

## Configure in LuCI

Open LuCI:

```text
Services → Tamizdat VPN
```

Recommended first setup:

1. **Servers (URIs)** — paste one or more `tamizdat://` profiles generated by the Tamizdat panel, then **Save & Apply**.
2. **Routing & Settings** — choose the interfaces to tunnel, for example `lan`.
3. **Routing & Settings** — click **Download / refresh geodata now** if you want geosite/geoip split routing.
4. **Routing & Settings** — choose the default policy:
   - proxy everything except direct rules;
   - or direct everything except proxy rules.
5. Add optional domain/geosite/geoip/IP rules and order them by priority. First match wins.
6. **Status** — click **Test all** to compare profile RTT through the protocol.
7. **Status** — click **Make active** on the preferred profile.
8. **Status** — **Enable & start**.

Domain/geosite rules require LAN clients to use the router for DNS, because dnsmasq fills nft sets as names resolve.

## CLI operation

Enable and start with the selected active profile:

```sh
uci set tamizdat.global.enabled=1
uci commit tamizdat
/etc/init.d/tamizdat restart
```

Select an active profile by UCI section name:

```sh
uci set tamizdat.global.active=<section>
uci commit tamizdat
/etc/init.d/tamizdat restart
```

View logs:

```sh
logread -e tamizdat
```

Refresh geodata and inspect nft state:

```sh
/usr/libexec/tamizdat/tamizdat-routing.sh geoupdate
/usr/libexec/tamizdat/tamizdat-routing.sh up
nft list table inet tamizdat
```

## Package/feed build mode

The archive can also be used as an OpenWrt feed source. Drop these directories into your feed and build with the OpenWrt SDK:

```sh
make package/tamizdat/compile
make package/luci-app-tamizdat/compile
```

For most router tests, the direct on-router `install.sh` path is faster and does not require building `.ipk` packages.

## Safety notes

- Treat full `tamizdat://` URIs as credentials.
- Do not paste real URIs into issue reports, screenshots, or logs.
- The shipped archive is aarch64-only. Other architectures need rebuilt binaries.
- IPv6 on tunneled interfaces is blocked by default in the OpenWrt routing helper to avoid leaking around the IPv4 tunnel. Change the LuCI routing setting only if you intentionally want native/direct IPv6.
